## 2. Детализация критерия №1 (Рейтинг)

### Уровень 1 — Первичный рейтинг

| Пункт из ТЗ | Статус | Подтверждение |
|---|---|---|
| Данные анкеты (возраст/пол/интересы/гео) | ✅ | `services/ranking-service/app/formulas.py` (`has_age`, `has_gender`, `has_city`, `interests_count`) |
| Полнота анкеты + количество фото | ✅ | `primary_score(...)` учитывает `completeness` и `photos_count` |
| Первичные предпочтения пользователя | ✅ | `has_preferences` в L1 + реальные фильтры `age/gender/city` в ленте (`services/ranking-service/app/feed_service.py`) |

### Уровень 2 — Поведенческий рейтинг

| Пункт из ТЗ | Статус | Подтверждение |
|---|---|---|
| Количество лайков | ✅ | `behavioral_score(... likes_received ...)` |
| Соотношение лайков/пропусков | ✅ | `ratio = likes / (likes + skips)` |
| Частота взаимных лайков (мэтчей) | ✅ | `mutual_matches` в формуле и SQL-агрегации в `tasks.py` |
| Частота начала диалогов после мэтча | ✅ | `dialogs_started` через `started_dialog_at` |
| Временная активность | ✅ | `active_hours_count` в формуле |

### Уровень 3 — Комбинированный рейтинг

| Пункт из ТЗ | Статус | Подтверждение |
|---|---|---|
| Интеграция L1 + L2 по весам | ✅ | `combined_score(primary, behavioral, ...)` |
| Реферальный фактор | ✅ | `referral_bonus` + cap в `combined_score(...)` и `tasks.py` |

Итог по критерию №1: `все пункты из каждого уровня закрыты` (уровень на 3 балла).

## 4. Рекомендации, чтобы защита прошла как «полностью соответствует»

1. Приложить короткий протокол ручного E2E прогона:
   - `/start` → регистрация → просмотр ленты → лайк/скип → мэтч → уведомление.
2. Один раз прогнать JMeter и сохранить:
   - `results.jtl`
   - `report/index.html`
   - 2–3 скриншота из Grafana во время нагрузки.
3. Обновить `SETUP.md`, чтобы не было противоречий между документацией и реальным состоянием кода.

## 6. Вывод

По коду и инфраструктуре проект закрывает основные технические критерии (`1–8`) и большую часть этапов (`9`).  
Главные незакрытые моменты для формального «полного соответствия» — это не реализация, а отсутствие приложенных подтверждений прогонов (ручной E2E и JMeter) и актуализация части документации.
